# 実装タスクリスト

## フェーズ 1: 環境構築 & スキャフォールディング (雛形作成)
- [ ] タスク: プロジェクトディレクトリ構成の構築と基本ファイルの作成 (Dockerfile, docker-compose.yml)
  - 完了条件: `docker compose up` でプロジェクトがビルド・起動可能になること。
  - バリデーションタスク: コンテナ内の Streamlit アプリが正しく起動し、ブラウザからアクセス可能であることを確認する。
- [ ] タスク: アプリケーションの基本雛形の実装 (src/main.py)
  - 完了条件: アプリケーションの構造が実装されており、プレースホルダー用のロジックが含まれていること。
  - バリデーションタスク: 構築されたアプリ構造が正しく反映されていることを確認する。

## フェーズ 2: コアロジック実装 (MVP)
- [ ] タスク: データハンドリングモジュールの実装 (`src/core/data_handler.py`)
  - 完了条件: CSVの読み込みとカラムリスト抽出が正常に動作すること。
  - バリデーションタスク: 様々な有効/無効なCSVファイルを用いて、pytest による単体テストで検証する。
- [ ] タスク: 統計エンジン実装 (`src/engine/statistics.py`)
  - 完了条件: ヒストグラムとQ-Qプロットの計算ロジックが正確に動作すること。
  - バリデーションタスク: 既知の統計値と比較して計算精度を検証する。
- [ ] タスク: 可視化マネージャー実装 (`src/visualization/plotter.py`)
  - 完了条件: グラフ（ヒストグラム、Q-Qプロット）が正しく生成されること。
  - バリデーションタスク: Streamlit UI上でのグラフ表示の目視確認と、プロットオブジェクト生成に関する単体テスト。
- [ ] タスク: コンポーネントの統合 (`src/main.py`)
  - 完了条件: アップロード -> カラム選択 -> グラフ表示 の一連のユーザーフローが完結すること。
  - バリデーションタスク: ユーザー視点での手動E2Eテストを実施する。

  - 完了条件: 全てのテストがパスし、製品として利用可能な状態になること。
