# 備品管理・貸出管理アプリ 要件定義書

## 1. 概要

### 1.1 アプリビジョン
社員による備品の借り出し・返却を一元管理し、現在の在庫状況や貸出状況を可視化するWebアプリケーション。

### 1.2 対応する業務課題

| ID | 業務課題 | 説明 |
|-----|---------|------|
| RQ-BK-001 | 備品の在庫管理の効率化 | 現在、手動で備品の在庫を管理しており、正確性や最新性に課題がある。システムで一元管理することにより、効率的な在庫把握を実現する。この課題がなければ、備品の所在が不明確になり、必要な備品を見つけられず業務が停滞する。 |
| RQ-BK-002 | 貸出状況の可視化 | 現在、誰がいつどの備品を借りているのかが把握しにくく、返却漏れや紛失のリスクがある。システムで貸出・返却を記録し、現在の貸出先を明確にすることにより、返却漏れ防止と紛失検出（貸出先への確認）を実現する。この課題がなければ、備品の紛失や返却遅延が増加し、組織の資産管理が成立しない。 |

### 1.3 プロジェクト前提条件

- 外部システムとの連携は不要
- オンプレミス環境での運用を想定
- Webアプリケーションとして実装

## 2. 業務エンティティと状態管理

### 2.1 業務エンティティ一覧

本システムで管理する主要なエンティティは以下の通り：

1. **備品** - 管理対象となる物品
2. **貸出記録** - 備品の貸出・返却履歴
3. **ユーザー/借用者** - システムの利用者（社員）

### 2.2 備品エンティティ

**RQ-DT-001**: 備品の管理属性

| 属性 | 説明 | 必須 |
|------|------|------|
| 一意識別子 | 資産管理番号（ユーザーが入力、重複チェック必須） | ○ |
| 名前 | 備品の名称 | ○ |
| 現在の状態 | 「利用可能」または「借出中」（貸出記録から動的に判定） | ○ |

**備品の状態遷移**
```
利用可能 <---> 借出中
  ↓貸出処理   ↓返却処理
```

- **利用可能**: 返却されている、または貸出記録が存在しない状態
- **借出中**: 返却日時が記録されていない貸出記録が存在する状態

### 2.3 貸出記録エンティティ

**RQ-DT-002**: 貸出記録の管理属性

| 属性 | 説明 | 必須 |
|------|------|------|
| 借用者 | ユーザーエンティティへの参照 | ○ |
| 備品 | 備品エンティティへの参照 | ○ |
| 貸出日時 | 備品を貸し出した日時 | ○ |
| 返却日時 | 備品を返却した日時（未返却の場合はNULL） | - |

**データ保持方針**

RQ-DT-004: 返却時にレコード削除
- 貸出記録は「現在の貸出状況」を把握するためのみ保持
- 返却完了時に当該貸出記録を削除
- 過去の履歴は保持しない

### 2.4 ユーザーエンティティ

**RQ-DT-003**: ユーザーの管理属性

| 属性 | 説明 | 必須 |
|------|------|------|
| 社員ID | 社員を一意識別する番号 | ○ |
| 名前 | 社員の氏名 | ○ |
| ユーザー名 | ログイン用ユーザー名 | ○ |
| パスワード | ログイン用パスワード（複雑さ要件なし） | ○ |
| 権限 | 「管理者」または「一般社員」 | ○ |

**権限の種類**

| 権限 | 説明 |
|------|------|
| 管理者 | 備品登録、貸出処理、返却処理、ユーザー管理、パスワード変更が可能 |
| 一般社員 | 備品一覧画面での閲覧のみ可能（貸出状態の確認） |

## 3. 業務機能要件

### 3.1 備品管理機能

**RQ-FT-001**: 備品の登録
- **対応業務課題**: RQ-BK-001
- **実行者**: 管理者のみ
- **処理内容**: 新しい備品をシステムに登録する
- **入力情報**: 資産管理番号、名前
- **検証**: 同じ資産管理番号で登録できない（重複チェック）
- **初期状態**: 登録時点で「利用可能」状態
- **必要画面**: RQ-UI-003（備品登録画面）

**RQ-FT-004**: 備品一覧の確認
- **対応業務課題**: RQ-BK-001、RQ-BK-002
- **実行者**: 全員（管理者・一般社員）
- **表示内容**: 全備品を以下の情報とともに表示
  - 資産管理番号
  - 備品名
  - 現在の貸出状態（利用可能/借出中）
  - 管理者の場合：返却ボタン（貸出中の備品のみ表示）
- **表示順序**: 備品名の昇順（五十音順）
- **検索機能**: なし（全備品を一覧表示）
- **必要画面**: RQ-UI-002（備品一覧画面）

### 3.2 貸出・返却機能

**RQ-FT-002**: 貸出処理
- **対応業務課題**: RQ-BK-002
- **実行者**: 管理者のみ
- **処理内容**: ユーザーが備品を借り出す際に記録
- **入力方法**:
  - 借用者: ドロップダウンでユーザー一覧から選択
  - 備品: 備品一覧から貸出対象の備品を選択（利用可能な備品のみ対象）
- **処理結果**: 貸出記録を作成（貸出日時は自動設定）
- **制限**: 1回の処理で1つの備品のみ貸出可能
- **必要画面**: RQ-UI-004（貸出処理画面）

**RQ-FT-003**: 返却処理
- **対応業務課題**: RQ-BK-002
- **実行者**: 管理者のみ
- **処理内容**: ユーザーが備品を返却する際に記録
- **実行方法**: 備品一覧画面で「借出中」の備品に表示される「返却ボタン」をクリック
- **処理内容**: 返却ボタンをクリックすると即座に返却を記録し、該当貸出記録を削除
- **確認画面**: なし
- **必要画面**: RQ-UI-002（備品一覧画面の返却ボタン）

### 3.3 ユーザー管理機能

**RQ-FT-005**: ユーザー（社員）の管理
- **対応業務課題**: RQ-BK-001、RQ-BK-002
- **実行者**: 管理者のみ
- **処理内容**: 社員ユーザーの登録、編集、削除を一元管理
- **画面構成**:
  - ユーザー一覧を表示
  - 一覧から「編集」「削除」ボタンで操作可能
  - 新規登録フォーム（名前、社員ID、ユーザー名、パスワード、権限）を同画面に配置
- **検索機能**: なし
- **削除処理**: 退職者はシステムから削除
- **必要画面**: RQ-UI-005（ユーザー管理画面）

## 4. 認証・認可機能

### 4.1 認証機能

**RQ-AU-001**: ログイン機能
- **対応業務課題**: RQ-BK-001、RQ-BK-002
- **認証方式**: ユーザー名とパスワードでのログイン
- **初期管理者**: アプリケーション初回インストール時に、デフォルト管理者を自動作成
  - ユーザー名: admin
  - パスワード: password
- **新規ユーザーのパスワード**: 管理者がユーザー管理画面で登録時に入力
- **パスワード複雑さ要件**: なし（シンプルなパスワードでOK）
- **必要画面**: RQ-UI-001（ログイン画面）

**RQ-AU-002**: パスワード変更機能
- **対応業務課題**: RQ-BK-001、RQ-BK-002
- **実行者**: ログイン済みの全ユーザー（自分のパスワード変更のみ）
- **入力方法**: 「現在のパスワード」と「新しいパスワード」の両方を入力
- **検証**: 現在のパスワードが正しいことを確認してから変更を適用
- **必要画面**: RQ-UI-006（パスワード変更画面）

**RQ-AU-003**: セッション管理
- **対応業務課題**: RQ-BK-001、RQ-BK-002
- **セッションタイムアウト**: 30分間操作がない場合、自動的にログアウト
- **ログアウト機能**: ヘッダーメニューにログアウトボタンを配置、クリックするとログイン画面に戻る

### 4.2 認可機能

**RQ-AC-001**: 管理者権限
- **対応業務課題**: RQ-BK-001、RQ-BK-002
- **実行可能操作**:
  - 備品の登録（RQ-FT-001）
  - 貸出処理（RQ-FT-002）
  - 返却処理（RQ-FT-003）
  - ユーザー管理（RQ-FT-005）
  - パスワード変更（RQ-AU-002）
- **画面アクセス**: 以下の全画面にアクセス可能
  - RQ-UI-002（備品一覧画面、返却ボタン表示）
  - RQ-UI-003（備品登録画面）
  - RQ-UI-004（貸出処理画面）
  - RQ-UI-005（ユーザー管理画面）
  - RQ-UI-006（パスワード変更画面）

**RQ-AC-002**: 一般社員権限
- **対応業務課題**: RQ-BK-001、RQ-BK-002
- **実行可能操作**:
  - 備品一覧の閲覧（RQ-FT-004）
  - パスワード変更（RQ-AU-002）
- **画面アクセス**: 以下の画面のみアクセス可能
  - RQ-UI-002（備品一覧画面、表示のみ）
  - RQ-UI-006（パスワード変更画面）

## 5. 画面要件

### 5.1 画面一覧

| 画面ID | 画面名 | 対応業務課題 | アクセス権限 | 説明 |
|--------|--------|------------|------------|------|
| RQ-UI-001 | ログイン画面 | RQ-BK-001、RQ-BK-002 | 全員 | ユーザー名とパスワードを入力してログイン |
| RQ-UI-002 | 備品一覧画面 | RQ-BK-001、RQ-BK-002 | 全員 | 全備品の一覧と貸出状態を表示。管理者のみ返却ボタン表示 |
| RQ-UI-003 | 備品登録画面 | RQ-BK-001 | 管理者 | 新しい備品を登録（資産管理番号、備品名） |
| RQ-UI-004 | 貸出処理画面 | RQ-BK-002 | 管理者 | ユーザーを選択し、借り出す備品を選択して貸出を記録 |
| RQ-UI-005 | ユーザー管理画面 | RQ-BK-001、RQ-BK-002 | 管理者 | 社員ユーザーの登録、編集、削除を管理 |
| RQ-UI-006 | パスワード変更画面 | RQ-BK-001、RQ-BK-002 | 全員 | ログイン中のユーザーが自分のパスワードを変更 |

### 5.2 共通UI要素

**RQ-UI-COM-001**: ヘッダーメニュー
- **配置**: 画面上部に固定
- **表示内容（管理者）**:
  - 備品一覧
  - 備品登録
  - 貸出処理
  - ユーザー管理
  - パスワード変更
  - ログアウトボタン
- **表示内容（一般社員）**:
  - 備品一覧
  - パスワード変更
  - ログアウトボタン
- **動作**: メニュー項目をクリックすると対応する画面に遷移

### 5.3 画面遷移フロー

```
ログイン画面（RQ-UI-001）
  ↓ ログイン成功
備品一覧画面（RQ-UI-002）
  ↓ メニューから選択
  ├→ 備品登録画面（RQ-UI-003）[管理者のみ]
  ├→ 貸出処理画面（RQ-UI-004）[管理者のみ]
  ├→ ユーザー管理画面（RQ-UI-005）[管理者のみ]
  ├→ パスワード変更画面（RQ-UI-006）[全員]
  └→ ログアウト → ログイン画面へ
```

## 6. 非機能要件

### 6.1 性能要件

**RQ-NF-001**: 利用規模
- **対応業務課題**: RQ-BK-001、RQ-BK-002
- **総利用人数**: 約100名（全社員）
- **同時アクセス最大数**: 10名

### 6.2 環境要件

**RQ-NF-002**: 運用環境
- **対応業務課題**: RQ-BK-001、RQ-BK-002
- **環境**: オンプレミス
- **アプリケーション形態**: Webアプリケーション（ブラウザで動作）
- **インストール**: 不要（Webサーバーで実行、クライアントはブラウザで接続）

### 6.3 セキュリティ要件

**RQ-NF-003**: 認証・認可
- **対応業務課題**: RQ-BK-001、RQ-BK-002
- **ログイン認証**: ユーザー名とパスワード（RQ-AU-001参照）
- **パスワード複雑さ要件**: なし
- **セッション管理**: 30分間操作がない場合は自動ログアウト

**RQ-NF-004**: ログ・バックアップ
- **操作ログ記録**: 不要
- **バックアップ**: 不要

## 7. テスト用利用シナリオ

### 7.1 利用シナリオ

**RQ-TS-001**: 管理者による備品登録から貸出・返却までの一連流れ
- **対応業務課題**: RQ-BK-001、RQ-BK-002
- **シナリオ**:
  1. 管理者がログイン
  2. 新しい備品を3つ登録（「プロジェクタA」「マイク」「HDMIケーブル」）
  3. 備品一覧で3つが「利用可能」状態で表示されることを確認
  4. 貸出処理画面で「社員Aが『プロジェクタA』を借り出す」を記録
  5. 備品一覧で「プロジェクタA」が「借出中」に変わり、返却ボタンが表示されることを確認
  6. 返却ボタンをクリックして返却を記録
  7. 備品一覧で「プロジェクタA」が「利用可能」に戻ることを確認

**RQ-TS-002**: 一般社員による備品一覧の閲覧
- **対応業務課題**: RQ-BK-001、RQ-BK-002
- **シナリオ**:
  1. 一般社員がログイン
  2. 備品一覧画面が表示される
  3. 全備品と貸出状態が見えることを確認
  4. 返却ボタン、備品登録、貸出処理、ユーザー管理のメニューが表示されていないことを確認
  5. パスワード変更画面にアクセス可能なことを確認

## 9. 削除候補の検討

以下の項目は、MVP要件から削除可能か検討した結果：

- **操作ログ記録**: 削除（監査が不要な場合の簡素化）
- **バックアップ**: 削除（初期段階では手動対応で対応可能）
- **パスワード複雑さ要件**: 削除（簡易システムのため、複雑さ不要）
- **セッションタイムアウト警告**: 削除（30分の自動ログアウトのみで対応）
- **備品検索機能**: 削除（初期段階では全一覧表示で対応）
- **初期スケジューリング機能**: 削除（MVP要件外）
- **メール通知機能**: 削除（MVP要件外）

**結論**: 削除した要件のみでも業務が成立することを確認。MVP要件として適切。

## 10. まとめ

本要件定義書は、インタビュー駆動の要件定義スキル（isdd）に基づき、ユーザーとの対話を通じて確定された要件をまとめています。

- **業務課題**: 2つ（RQ-BK-001、RQ-BK-002）
- **業務エンティティ**: 3つ（備品、貸出記録、ユーザー）
- **業務機能**: 5つ（備品登録、備品一覧、貸出処理、返却処理、ユーザー管理）
- **画面**: 6つ（ログイン、備品一覧、備品登録、貸出処理、ユーザー管理、パスワード変更）
- **非機能要件**: 性能、環境、セキュリティを網羅

全ての要件に「RQ-*」というIDを付与し、業務課題との対応を明確にしています。
