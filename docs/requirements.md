# 要件定義書：検査データ分布比較システム（MVP）

## 1. 目的・前提

### システム目的
製造工程における品質管理プロセスにおいて、異なる測定データセット間の分布を視覚的かつ定量的に比較できるようにすること。これにより、エンジニアが原因特定と工程最適化を効率よく行い、結果として製造コスト削減に貢献することがシステム全体の目標とする業務課題解決です。

### 用語集
*   **シグマプロット**: 本ドキュメントではQ-Qプロット（Quantile–Quantile Plot）のことであり、2つのデータセットの分布形状を同一スケールで比較するためのグラフです。

## 2. 業務要件

### 対象業務一覧
製造プロセスにおける工程間での品質管理データの確認および分布比較作業。

### 利用者・担当者
品質管理部門のエンジニアが主要な利用者となります。

### 業務課題とKPI、対応方針
*   **解決すべき業務課題**: 製造工程ごとに取得される測定値データについて、「どの程度のばらつきや偏りがあるか」「あるべき分布から外れていないか」を一目で判断することが難しい点。
*   **KPI (重要業績評価指標)**: 分布比較およびデータ分析にかかる時間短縮。
*   **対応方針**: データアップロードとGUI操作を極力簡素化し、必要な統計情報（Q-Qプロット、ヒストグラム）を即座に提示することで、エンジニアの分析時間を大幅に削減します。

## 3. 機能要件

本システムはWeb GUIベースで動作することを前提とします。

### 機能一覧
1.  **CSVデータアップロード機能**: エンジニアがローカルのCSVファイルをアプリケーションへアップロードできること。
2.  **比較対象カラム選択機能**: アップロードされたデータセットから、ユーザーが必要とする任意の2つの数値測定値を持つカラムを選択し、比較設定ができること。
3.  **分布比較表示機能**: 以下の視覚的・統計的な要素を同時にGUI上に描画すること。
    *   Q-Qプロット（シグマプロット）による分布の形状比較。
    *   それぞれのデータセットに対応するヒストグラム表示。

### データ仕様 (入力)
*   **形式**: CSVファイル形式。
*   **内容**: 複数の数値測定値を含む単一のCSVファイルを想定します。
*   **規模**: 最大数千レコード（データの量）を処理できること。

### 出力データ・外部連携
*   本システムは解析結果や元データの永続的な保存は行わないため、出力するデータ仕様はありません。また、外部システムとの連携機能は要求しません。

## 4. データ要件
本システムのMVPフェーズにおいては、解析結果や入力データの一時的な保持のみに留まり、永続的に管理・保存すべき業務エンティティ（マスターデータ含む）は存在しないため、該当するデータ要件はありません。

## 5. 非機能要件
*   **性能**: CSVファイルのアップロード完了から、すべてのグラフ描画が完了するまでの応答時間は10秒以内を目標とします。（最重要）
*   **利用人数・アクセス制限**: 同時接続ユーザーは単独での利用（一人）を想定し、特別な認証機能は要求しません。
