# 備品管理・貸出管理アプリ 実装タスク化

## タスク一覧

### Phase 1: 基盤構築

#### TASK-001: データベース初期化
- **対応設計**: DS-DB-001～006
- **対応要件**: RQ-DT-001～004
- **説明**: 4テーブル(users, equipments, loans, sessions)を作成し、初期管理者ユーザー(admin/password)を作成
- **実装内容**:
  - users テーブル作成（user_id, employee_id, name, username, password, role, created_at, updated_at）
  - equipments テーブル作成（equipment_id, asset_number, name, created_at, updated_at）
  - loans テーブル作成（loan_id, equipment_id, user_id, loaned_at）
  - sessions テーブル作成（session_id, user_id, created_at, last_activity, ip_address）
  - UNIQUE制約・外部キー制約・インデックス設定
  - 初期管理者自動作成機能
- **成果物**: 初期化スクリプト（init_db.py等）

#### TASK-002: プロジェクト構成・フレームワーク選定
- **対応設計**: DS-ARC-001, DS-ARC-002
- **対応要件**: RQ-NF-002
- **説明**: Webアプリケーションフレームワーク(Flask/Express等)を選定し、プロジェクト構造を構築
- **実装内容**:
  - フレームワーク選定（推奨: Python Flask）
  - ディレクトリ構造設計（app/, templates/, static/, etc）
  - 依存ライブラリ管理（requirements.txt等）
  - 基本的なアプリケーション初期化
- **成果物**: プロジェクトスケルトン

### Phase 2: 認証・認可機能

#### TASK-003: ログイン機能実装
- **対応設計**: DS-API-001, DS-UI-001, DS-FLOW-001, DS-SEC-001
- **対応要件**: RQ-AU-001, RQ-FT-005
- **説明**: ユーザー認証を実装し、セッション生成・Cookie設定を行う
- **実装内容**:
  - POST /api/auth/login エンドポイント実装
  - ユーザー名とパスワードによる認証ロジック
  - bcrypt パスワード比較
  - セッションID生成・sessions テーブル保存
  - HttpOnly/Secure Cookie設定
  - ログイン画面(RQ-UI-001)実装：HTML/CSS/JavaScriptフォーム
  - バリデーション（ユーザー名・パスワード必須）
  - エラーメッセージ表示
- **成果物**: ログイン API・画面

#### TASK-004: セッション管理・タイムアウト実装
- **対応設計**: DS-DB-004, DS-API-012, DS-FLOW-004, DS-SEC-001
- **対応要件**: RQ-AU-003
- **説明**: セッション検証とタイムアウト自動検出機能を実装
- **実装内容**:
  - 全API呼び出し前のセッション検証ミドルウェア実装
  - 30分タイムアウト判定ロジック
  - last_activity 自動更新
  - タイムアウト時のセッションレコード削除
  - 401エラー返却・クライアント側ログイン画面リダイレクト
- **成果物**: セッション管理ミドルウェア

#### TASK-005: ログアウト機能実装
- **対応設計**: DS-API-002, DS-UI-002, DS-SEC-001
- **対応要件**: RQ-AU-003
- **説明**: ログアウト処理とセッション削除を実装
- **実装内容**:
  - POST /api/auth/logout エンドポイント実装
  - sessions テーブルからレコード削除
  - Cookie削除（クライアント側）
  - ログイン画面へリダイレクト
- **成果物**: ログアウト API

#### TASK-006: パスワード変更機能実装
- **対応設計**: DS-API-003, DS-UI-006, DS-SEC-001
- **対応要件**: RQ-AU-002, RQ-AC-001, RQ-AC-002
- **説明**: ユーザー本人がパスワードを変更できる機能を実装
- **実装内容**:
  - POST /api/auth/change-password エンドポイント実装
  - 現在のパスワード検証
  - 新パスワード・確認パスワード一致確認
  - bcrypt で新パスワード ハッシュ化・保存
  - パスワード変更画面(RQ-UI-006)実装
  - エラーメッセージ処理
  - 変更後ログイン画面へリダイレクト
- **成果物**: パスワード変更 API・画面

#### TASK-007: 権限チェック機能実装
- **対応設計**: DS-SEC-002, DS-API-001～011
- **対応要件**: RQ-AC-001, RQ-AC-002, RQ-NF-003
- **説明**: 全API呼び出しで権限チェックを実装
- **実装内容**:
  - 管理者チェックデコレータ/ミドルウェア実装
  - 管理者限定API前処理
  - 権限不足時の403エラー返却
  - 画面表示時の権限別メニュー表示制御
- **成果物**: 権限チェック共通処理

### Phase 3: 備品管理機能

#### TASK-008: 備品登録API実装
- **対応設計**: DS-API-005, DS-DB-002
- **対応要件**: RQ-FT-001, RQ-DT-001
- **説明**: 新しい備品をシステムに登録するAPI を実装
- **実装内容**:
  - POST /api/equipments エンドポイント実装
  - 管理者権限チェック
  - asset_number 重複チェック
  - 必須フィールドバリデーション
  - equipments テーブルにINSERT
  - 成功時201レスポンス
- **成果物**: 備品登録 API

#### TASK-009: 備品登録画面実装
- **対応設計**: DS-UI-003
- **対応要件**: RQ-FT-001, RQ-UI-003
- **説明**: 管理者向けの備品登録画面を実装
- **実装内容**:
  - 備品登録画面HTML/CSS/JavaScript
  - 資産管理番号・備品名入力フィールド
  - バリデーション（必須チェック）
  - 登録ボタン（POST /api/equipments 呼び出し）
  - キャンセルボタン（備品一覧へ）
  - 成功/エラーメッセージ表示
  - 2秒後に備品一覧へリダイレクト
- **成果物**: 備品登録画面

#### TASK-010: 備品一覧取得API実装
- **対応設計**: DS-API-004, DS-DB-002, DS-DB-003
- **対応要件**: RQ-FT-004, RQ-DT-001
- **説明**: 全備品を一覧取得し、貸出状態を判定して返すAPI を実装
- **実装内容**:
  - GET /api/equipments エンドポイント実装
  - 全権限でアクセス可能
  - equipments テーブル全件取得
  - 備品名の昇順ソート
  - 各備品について loans テーブルをチェック
    - レコードあり → "borrowed", 借用者名も返す
    - レコードなし → "available"
  - JSON形式で返却
- **成果物**: 備品一覧 API

#### TASK-011: 備品一覧画面実装
- **対応設計**: DS-UI-002, DS-UI-COM-001
- **対応要件**: RQ-FT-004, RQ-UI-002
- **説明**: 全ユーザーが備品を確認できる一覧画面を実装
- **実装内容**:
  - 備品一覧画面HTML/CSS/JavaScript
  - ヘッダーメニュー実装（権限別表示）
    - 管理者: 備品一覧, 備品登録, 貸出処理, ユーザー管理, パスワード変更, ログアウト
    - 一般社員: 備品一覧, パスワード変更, ログアウト
  - テーブル形式で備品表示（資産番号, 備品名, 状態）
  - 返却ボタン（管理者かつ借出中のみ表示）
  - 初期化時に GET /api/equipments 呼び出し
  - 定期更新（5秒ごと）
  - 権限に応じた操作ボタン表示制御
- **成果物**: 備品一覧画面・ヘッダーメニュー

#### TASK-012: 返却ボタン実装
- **対応設計**: DS-UI-002, DS-API-007
- **対応要件**: RQ-FT-003, RQ-UI-002
- **説明**: 備品一覧画面の返却ボタンを実装
- **実装内容**:
  - 返却ボタン（借出中の備品行にのみ表示）
  - クリック時に DELETE /api/loans/{equipment_id} 呼び出し
  - 確認画面なし（即座に実行）
  - 成功時：テーブルリロード、該当行の状態を「利用可能」に変更
  - 失敗時：エラーメッセージ表示
- **成果物**: 返却ボタン機能

### Phase 4: 貸出機能

#### TASK-013: 貸出処理API実装
- **対応設計**: DS-API-006, DS-DB-003
- **対応要件**: RQ-FT-002, RQ-DT-002
- **説明**: 備品を貸し出すAPI を実装
- **実装内容**:
  - POST /api/loans エンドポイント実装
  - 管理者権限チェック
  - equipment_id, user_id 存在確認
  - loans テーブルに当該 equipment_id のレコードがないか確認
  - 既に借出中の場合は400エラー
  - 新規貸出記録をINSERT（loaned_at は現在時刻）
  - 成功時201レスポンス
- **成果物**: 貸出処理 API

#### TASK-014: 貸出処理画面実装
- **対応設計**: DS-UI-004
- **対応要件**: RQ-FT-002, RQ-UI-004
- **説明**: 管理者向けの貸出処理画面を実装
- **実装内容**:
  - 貸出処理画面HTML/CSS/JavaScript
  - 借用者ドロップダウン（全ユーザーを表示、「社員ID - 名前」形式）
  - 備品ドロップダウン（利用可能な備品のみを表示）
  - 貸出ボタン（POST /api/loans 呼び出し）
  - キャンセルボタン（備品一覧へ）
  - バリデーション（借用者・備品選択必須）
  - エラー表示（既に借出中等）
  - 成功時2秒後に備品一覧へリダイレクト
- **成果物**: 貸出処理画面

#### TASK-015: 返却処理API実装
- **対応設計**: DS-API-007, DS-DB-003, DS-FLOW-003
- **対応要件**: RQ-FT-003, RQ-DT-004
- **説明**: 備品を返却するAPI を実装
- **実装内容**:
  - DELETE /api/loans/{equipment_id} エンドポイント実装
  - 管理者権限チェック
  - loans テーブルで該当 equipment_id のレコード検索
  - レコードが存在しない場合は404エラー
  - レコードをDELETE
  - 成功時200レスポンス
- **成果物**: 返却処理 API

#### TASK-016: ユーザー一覧取得API実装
- **対応設計**: DS-API-008, DS-DB-001
- **対応要件**: RQ-FT-005
- **説明**: 全ユーザーを一覧取得するAPI を実装

  - 管理者権限チェック

  - JSON形式で返却（パスワードハッシュは除外）
- **成果物**: ユーザー一覧 API

#### TASK-017: ユーザー登録API実装
- **対応設計**: DS-API-009, DS-DB-001
- **対応要件**: RQ-FT-005, RQ-DT-003
- **説明**: 新規ユーザーを登録するAPI を実装
- **実装内容**:
  - POST /api/users エンドポイント実装
  - 管理者権限チェック
  - employee_id, username の重複チェック
  - 必須フィールド確認（employee_id, name, username, password, role）
  - パスワードを bcrypt でハッシュ化

- **成果物**: ユーザー登録 API

#### TASK-018: ユーザー更新API実装
- **対応設計**: DS-API-010, DS-DB-001
- **対応要件**: RQ-FT-005
- **説明**: 既存ユーザーを更新するAPI を実装
- **実装内容**:
  - PUT /api/users/{user_id} エンドポイント実装
  - 管理者権限チェック
  - ユーザー存在確認
  - username, employee_id の重複チェック（自分以外）
  - users テーブルをUPDATE
  - 成功時200レスポンス
- **成果物**: ユーザー更新 API

#### TASK-019: ユーザー削除API実装
- **対応設計**: DS-API-011, DS-DB-001
- **対応要件**: RQ-FT-005
- **説明**: ユーザーを削除するAPI を実装
- **実装内容**:
  - DELETE /api/users/{user_id} エンドポイント実装
  - 管理者権限チェック
  - ユーザー存在確認
  - admin ユーザーの削除を禁止
  - 該当ユーザーが貸出中の備品を持たないか確認（loans テーブル）
  - users テーブルからDELETE
  - 成功時200レスポンス
- **成果物**: ユーザー削除 API

#### TASK-020: ユーザー管理画面実装
- **対応設計**: DS-UI-005
- **対応要件**: RQ-FT-005, RQ-UI-005
- **説明**: 管理者向けのユーザー管理画面を実装
- **実装内容**:
  - ユーザー管理画面HTML/CSS/JavaScript
  - ユーザー一覧テーブル（社員ID, 名前, ユーザー名, 権限）
  - 編集ボタン（クリック時にフォームに値を填入）
  - 削除ボタン（削除確認ダイアログ表示、admin は削除不可）
  - 新規登録フォーム（社員ID, 名前, ユーザー名, パスワード, 権限）
  - 登録/更新ボタン
  - クリアボタン
  - 初期化時に GET /api/users 呼び出し
  - 登録/更新/削除後にテーブルをリロード
- **成果物**: ユーザー管理画面

#### TASK-024: 統合テスト実施

    2. 備品一覧表示確認
    3. 管理者メニューが非表示か確認
    4. パスワード変更画面アクセス可能か確認

  - 権限チェックテスト
- **成果物**: テスト結果報告書
